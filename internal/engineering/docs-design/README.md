# Internal Documentation

This folder contains internal design documents, contracts, and developmental artifacts. These are for Guardian developers and maintainers, not end users.

---

## Contents

### `design/`
Product philosophy, authority model, and design decisions made during development.

**Audience:** Developers, product stakeholders  
**Use:** Understanding Guardian's design rationale

### `contracts/`
Behavior contracts that lock down Guardian's quality guarantees. Any regression breaks CI.

**Audience:** QA, developers  
**Use:** Understanding what Guardian promises and how it's enforced

### `phases/`
Phase-by-phase development documentation from earlier releases.

**Audience:** Historians, contributors  
**Use:** Understanding how Guardian evolved

### `examples/`
Demo code and example JSON payloads used in development and testing.

**Audience:** Developers  
**Use:** Reference implementations
